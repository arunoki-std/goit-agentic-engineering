# reviewer-core/ insights

Findings specific to `@devdigest/reviewer-core` (pure review engine).

---

## What belongs here

- Gotchas with the grounding gate, map-reduce strategy, or structured output repair
- Non-obvious behaviors of prompt assembly (INJECTION_GUARD, wrapUntrusted, slot ordering)
- Decisions about the engine's public API or how course lesson slots are wired
- Surprises about how the engine behaves with different LLM providers or model outputs

## What does NOT belong here

- Things derivable from reading the code or README
- Standard TypeScript or LLM API practices
- In-progress or volatile state
- Cross-module findings (add those to the root insights/INSIGHTS.md)
