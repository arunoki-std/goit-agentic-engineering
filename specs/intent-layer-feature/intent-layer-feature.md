## Будуємо Intent Layer
Це і є та «справжня фіча», яку ми проганяємо через щойно зібраний флоу агентів: спершу збираємо набір, а потім його ж руками будуємо фічу. Сьогодні додаємо рев'юеру розуміння наміру створення PR. Працюємо разом, паралельно з відео — і будуємо через harness, самі при цьому використовуючи subagents з теорії.

## Що будуємо (скоуп фічі):
Нам треба збудувати фічу, яка називається "Intent Layer".
Всю інформацію по ній ти зможеш знайти у тому тексті, який я тобі дав вище. І також на дизайні ти можеш побачити цей блок "Intent Layer".
В цілому нам треба буде зрозуміти мотивацію, за якою відкрився даний pull request. Нам треба буде взяти дешеву модель, також додати ці опції вибору моделі в налаштування. І треба буде також із опису pull request взяти номер тікета, специфікацію, на основі якої була створена ця задача, або план робіт, де буде вказана мотивація. І цю мотивацію разом зі змінами треба буде використати для того, щоб реалізувати цей "Intent Layer".

- Класифікатор наміру: окремий виклик дешевої моделі (flash-клас через OpenRouter) зі structured-схемою Intent {summary, in_scope[], out_of_scope[]}. Вхід — PR title + body + linked issue + список файлів з hunk headers. Без тіл змін — і ми порахуємо в логах, скільки токенів це заощадило.

- Збереження: intent зберігається per-PR, перерахунок — за кнопкою (PR оновився → намір міг змінитися).

- Інжекція у рев'ю: блок intent додається у prompt агента з правилом «не коментуй поза intent; якщо бачиш серйозну проблему поза скоупом — одна знахідка-сигнал, не двадцять».

- UI: intent-картка на сторінці PR — ви бачите, як машина зрозуміла намір, до того, як читати рев'ю.
