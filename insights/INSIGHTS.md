# Project-level insights

Findings that apply across the whole project (not specific to one module).
Module-specific insights live in each module's own `insights/INSIGHTS.md`.

---

## What belongs here

- Cross-cutting gotchas (affect multiple packages simultaneously)
- Non-obvious interactions between modules discovered during work
- Architectural decisions made during course lessons and why
- Surprises about how tsconfig path aliases behave in practice

## What does NOT belong here

- Things derivable from reading the code
- Standard Node/TypeScript practices
- In-progress work or temporary state
- Per-module specifics (add those to the module's INSIGHTS.md instead)
