     Plan: Conventions Extractor Quality Improvement

     Context

     The Conventions Extractor already works end-to-end: it samples files, calls
     GPT-4.1, validates evidence deterministically, and persists candidates.
     The core UX (UI, routes, skill creation) stays untouched.

     Root cause of false negatives: getConventionSamples(repoId, 12) →
     getTopFilesByRank(repoId, 12) returns 12 files sorted by PageRank DESC.
     PageRank rewards import-hub files (src/platform/container.ts, shared types,
     adapters). In practice, all 12 slots come from 1-2 infrastructure
     directories. Feature module files (src/modules/reviews/service.ts,
     src/modules/agents/service.ts) rank lower and are never shown to the LLM →
     patterns like "workspace-scoped repository methods", "throw NotFoundError
     vs throw new Error", "early-return guards" are missed.

     Selected improvement: Directory-stratified sample selection — no extra LLM
     call, deterministic, small change.

     What to implement

     1. server/src/modules/conventions/service.ts

     Add two module-scoped helpers (after the existing helpers block, before
     ConventionsService):

     dirGroup(filePath: string): string — private, not exported
     - Split path by /
     - Strip filename: parts.slice(0, -1) → dirParts
     - If dirParts.length === 0 → return 'root'
     - Return dirParts.slice(0, 3).join('/') (cap at 3 segments to prevent
     over-splitting deep paths)

     Examples: src/modules/reviews/service.ts → src/modules/reviews;
     src/adapters/llm/x/y.ts → src/adapters/llm; index.ts → root

     export function diversifyPaths(paths: string[], n: number): string[] —
     exported for testability
     - If paths.length <= n → return paths unchanged
     - Build Map<string, string[]> grouping by dirGroup(), preserving input
     order within groups
     - If groups.size <= 1 → return paths.slice(0, n) (fallback: rank order)
     - Round-robin across groups via queue.shift() until result has n entries or
     all queues empty

     Change in extract() (line 319):

     Test cases:
     1. paths.length < n → returns unchanged
     2. paths.length === n → returns unchanged
     3. All paths in one group → returns top-n in rank order
     4. Two groups, n < total → round-robins: first from each group, then second
     from each, etc.
     5. Three groups evenly distributed → each group contributes equally
     6. Three groups, one with fewer files → small group exhausts, larger groups
     fill remaining slots
     7. Deep paths (4+ segments) collapse to 3-segment group key
