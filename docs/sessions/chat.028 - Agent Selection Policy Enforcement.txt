     Development Plan: Agent Selection Policy Enforcement

     Execution mode: main

     Context

     The planner agent currently has a wave/checkpoint protocol but no explicit
     rule governing when to use a main agent, a single implementer, or parallel
     implementers. As a result, planners over-allocate ‚ÄĒ creating
     multi-implementer waves for single-owner tasks. The completion-reviewer
     also lacks an explicit check for execution-mode mismatch.

     This task adds an Agent Selection Policy section to planner.md and a
     corresponding execution-mode verification check to completion-reviewer.md,
     then exposes the declared mode as a required field in the plan output
     template.

     Scope: 2 agent config files, ~25 lines added. No implementer needed ‚ÄĒ main
     agent applies directly.

     Files

     - .claude/agents/planner.md
     - .claude/agents/completion-reviewer.md

     Implementation Steps

     1. Add ## Agent Selection Policy to planner.md

     Insert a new section between ## Planning Method (line 73) and ## Validation
     Policy (line 75).

     Content:

     ## Agent Selection Policy

     Classify the task before assigning execution resources. Apply the first
     matching threshold:

     | Threshold | Execution mode |
     |---|---|
     | Mechanical change: ‚Č§ 5 files, ‚Č§ 10 new lines, no complex logic or tests |
     `main` ‚ÄĒ orchestrator applies directly; no implementer |
     | One cohesive task in one module | `single implementer` ‚ÄĒ one isolated
     worktree |
     | Two or more substantial, independent, disjoint outcomes | `parallel
     implementers` ‚ÄĒ one worktree per outcome |
     | Shared cross-module wiring after parallel waves | `integrator` ‚ÄĒ runs in
     main worktree after checkpoint merge |

     Enforcement rules:
     - A task with a single owner is always `single implementer`, never
     `parallel implementers`, even if it spans multiple files in that module.
     - A Checkpoint is required only when a later wave genuinely depends on the
     output of the earlier one. Do not add checkpoints between independent
     waves.
     - Do not create empty or trivial implementer waves (no wave whose only
     content is a config edit or a one-line change).
     - The plan output must include an explicit `Execution mode:` field (see
     Required Output).

     Where exactly: insert after the blank line following step 7 (line 73) and
     before the ## Validation Policy heading (line 75).

     2. Add Execution mode: field to the ## Required Output template in 
     planner.md

     After the # Development Plan: <short title> title line (line 101), add:

     **Execution mode:** `main` | `single implementer` | `parallel implementers`
     | `integrator`

     Choose exactly one value. This line must appear in every plan the planner
     produces.

     3. Extend step 7 in completion-reviewer.md with execution-mode verification

     Current step 7 (line 39) ends with: ... Flag overlapping planning/discovery
     ownership as P2. Report measured time/context only when explicitly 
     present.

     Append to that sentence:

     Also verify that the plan's declared `Execution mode` matches actual scope:
     flag P1 if parallel implementers were used for a single-owner task; flag
     P1 if a checkpoint is missing between genuinely dependent waves or present
     between independent ones; flag P2 if a trivial or empty implementer wave
     was created; flag P2 if the plan omits the `Execution mode:` field
     entirely.

     Verification

     After applying:
     1. Open .claude/agents/planner.md ‚ÄĒ confirm the new ## Agent Selection 
     Policy section appears between ## Planning Method and ## Validation Policy,
     and the Execution mode: line appears in the output template.
     2. Open .claude/agents/completion-reviewer.md ‚ÄĒ confirm step 7 now includes
     the execution-mode mismatch checks with correct severity labels.
     3. Mentally apply acceptance examples from the spec:
       - Two-file agent-config change ‚Üí plan declares main, reviewer passes
     without wave check.
       - One server feature ‚Üí plan declares single implementer, one worktree
     launched.
       - Independent client/server outcomes ‚Üí plan declares parallel 
     implementers, two worktrees with disjoint scopes.
       - Shared wiring ‚Üí plan declares integrator after checkpoint.
       - Empty wave ‚Üí reviewer flags P2.

     Definition of Done

     - [ ] planner.md contains ## Agent Selection Policy with the four-row
     threshold table and enforcement rules
     - [ ] planner.md output template requires Execution mode: field
     - [ ] completion-reviewer.md step 7 checks execution-mode match with P1/P2
     severities
     - [ ] No other files modified
