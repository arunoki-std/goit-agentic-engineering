     Plan: Replace Stop hook with explicit /wrap-up command

     Context

     The Stop hook in .claude/settings.json fires on every session end —
     micro-exchanges, plan-mode turns, Q&A — injecting a system message that
     forces the engineering-insights skill to run. Two concrete failure modes:

     1. Noise: The skill runs when nothing non-trivial happened, producing
     either empty writes or unwanted entries.
     2. Patch conflicts: Implementer agents run inside worktrees that inherit
     the same hook. When an implementer stops, the hook fires in the worktree
     context and writes to a stale snapshot of INSIGHTS.md. When the
     orchestrator later runs git apply on the feature branch, the patch fails
     because the branch's INSIGHTS.md already has newer entries (documented in
     INSIGHTS.md [2026-06-29]).

     The fix: remove the automatic hook, add an explicit /wrap-up command as the
     single INSIGHTS.md writer per session, and harden the implementer agent to
     return candidates rather than ever calling the skill directly.
