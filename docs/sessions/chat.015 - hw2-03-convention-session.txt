     Plan: Backend API ‚ÄĒ Convention Skill Preview & Create

     Context

     The conventions module already extracts and stores convention candidates
     from repos, and users can accept/reject them via PATCH. Now we need two new
     endpoints that turn those accepted candidates into a Skill record
     (source='extracted') that a reviewer agent can use. Agent linking is
     optional but supported via the existing AgentsService.linkSkill()
     mechanism.

     ---
     New Endpoints

     POST /repos/:id/conventions/skill-preview

     Stateless ‚ÄĒ builds and returns the markdown and metadata without persisting
     anything.

     Request body:
     { candidate_ids?: string[] }  // if omitted ‚Üí use all accepted for repo

     Response:
     {
       name: string;         // "{owner}/{name} Conventions"
       description: string;  // "Convention rules extracted from {owner}/{name}"
       type: "convention";
       enabled: true;
       body: string;         // full markdown (see format below)
       token_count: number;
     }

     Returns 422 if there are no accepted candidates (after applying the
     candidate_ids filter).

     ---
     POST /repos/:id/conventions/skill

     Creates the skill in the DB and optionally links it to an agent.

     Request body:
     {
       name: string;
       description: string;
       type: "rubric" | "convention" | "security" | "custom";
       enabled: boolean;
       body: string;
       agent_id?: string;  // UUID ‚ÄĒ if present, links skill to agent
     }

     Response: Full Skill DTO (same shape as GET /skills/:id).

     When agent_id is provided: calls AgentsService.linkSkill(workspaceId, 
     agentId, skill.id). Returns 404 if the agent doesn't exist.

     ---
     Markdown Format

     Grouped by category (null ‚Üí "General"). Each rule gets evidence for
     traceability.

     # owner/repo Conventions

     Flag changes that violate these rules and cite file:line when raising an
     issue.

     ## naming

     - Use camelCase for variable names

       Evidence: `src/utils/format.ts:12`
       const formattedValue = ...

     ## formatting

     - Always use trailing commas in multi-line objects

     Evidence: `src/config/index.ts:8`
       const config = {


     Rejected candidates are never included ‚ÄĒ only rows where accepted = true.

     evidenceFiles stored on the skill = unique non-null evidencePath values
     across accepted candidates.
