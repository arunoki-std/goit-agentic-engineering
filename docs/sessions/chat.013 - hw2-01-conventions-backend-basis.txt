     Plan: Conventions Extractor — Backend Foundation

     Context

     DevDigest HW2 requires a Conventions Extractor feature. This plan covers
     only the backend
     foundation: schema, migration, shared contract, module
     (repository/service/routes), and tests.
     No LLM extraction and no UI in scope.

     ---
     Step 1 — Extend conventions schema

     File: server/src/db/schema/knowledge.ts

     The existing table is missing three columns needed by the feature.
     Add them to the pgTable definition:

     import { integer } from 'drizzle-orm/pg-core'; // add to existing import

     // new columns inside the pgTable call:
     category:     text('category'),                                         // 
     nullable
     evidenceLine: integer('evidence_line'),                                 // 
     nullable
     createdAt:    now(),                                                     //
     timestamptz, default NOW(), NOT NULL

     All three are safe to add:
     - category / evidenceLine are nullable → no impact on existing rows
     - createdAt uses defaultNow() → existing rows get the migration timestamp

     ---
     Step 2 — Generate + apply migration

     cd server && pnpm db:generate   # produces 0010_conventions_extended.sql 
     (or similar)
     cd server && pnpm db:migrate

     The generated SQL will be:
     ALTER TABLE "conventions"
       ADD COLUMN "category" text,
       ADD COLUMN "evidence_line" integer,
       ADD COLUMN "created_at" timestamptz DEFAULT now() NOT NULL;

     Never edit the generated file; if the output looks wrong, fix the schema
     and regenerate.

     ---
     Step 3 — Update shared contract

     File: server/src/vendor/shared/contracts/knowledge.ts

     Update the ConventionCandidate Zod schema to add the two new nullable
     fields:

     export const ConventionCandidate = z.object({
       id:               z.string(),

     ---
     Step 4 — Create server/src/modules/conventions/

     4a. repository.ts

     Pattern: mirrors skills/repository.ts. All queries filter by workspaceId.

     export type ConventionRow = typeof t.conventions.$inferSelect;

     export class ConventionsRepository {
       constructor(private db: Db) {}

       async list(workspaceId: string, repoId: string): Promise<ConventionRow[]>
         // SELECT * FROM conventions WHERE workspace_id = $1 AND repo_id = $2 
     ORDER BY created_at DESC

       async getById(workspaceId: string, id: string): Promise<ConventionRow |
     undefined>
         // SELECT … WHERE workspace_id = $1 AND id = $2

       async update(workspaceId: string, id: string, patch: { accepted?:
     boolean; rule?: string }): Promise<ConventionRow | undefined>
         // UPDATE … WHERE workspace_id = $1 AND id = $2 RETURNING
     }

     4b. service.ts

     export class ConventionsService {
       private repo: ConventionsRepository;
       constructor(container: Container) { this.repo = new
     ConventionsRepository(container.db); }

       async list(workspaceId: string, repoId: string):
     Promise<ConventionCandidate[]>
       async update(workspaceId: string, repoId: string, id: string, patch):
     Promise<ConventionCandidate | undefined>
     }

     toDto(row) helper maps DB row → ConventionCandidate shape (snake_case).

     4c. routes.ts

     Two custom param schemas (can't reuse IdParams alone because PATCH has two
     UUIDs):

     const RepoParams     = z.object({ id: z.string().uuid() });
     const ConventionParams = z.object({ id: z.string().uuid(), conventionId:
     z.string().uuid() });
     const PatchBody      = z.object({ accepted: z.boolean().optional(), rule:
     z.string().min(1).optional() });

     Routes:
     GET  /repos/:id/conventions
     PATCH /repos/:id/conventions/:conventionId

     Both call getContext() first for workspace scoping.
     PATCH returns 404 (NotFoundError) when the convention doesn't exist or
     belongs to another workspace.
