     Parallel Implementer agents — proof-of-concept

     Context

     The user wants to verify that Claude Code can run multiple Implementer
     agents truly in parallel. As a vehicle for the test we'll use three
     trivial, independent JSDoc additions — one per package (server, client,
     reviewer-core) — so the agents touch completely disjoint files and can
     never conflict. The goal is not the docs themselves but observing parallel
     execution.

     What "parallel" means here

     All three Agent tool calls must be emitted in a single assistant message.
     The harness launches them concurrently; wall-clock time ≈ max(one agent)
     rather than sum(three agents). Each agent will get a task notification when
     it finishes; we'll see all three notifications arrive close together.

     Tasks (one per agent, zero overlap)
