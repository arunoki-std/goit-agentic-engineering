     План: Перехід з CLAUDE.md на AGENTS.md

     Context

     Проєкт DevDigest має 5 файлів CLAUDE.md (корінь + server/ client/
     reviewer-core/ e2e/). Користувач хоче, щоб інші AI-інструменти (Codex,
     Cursor, Gemini тощо) теж читали інструкції з проєкту. Ці інструменти
     шукають файл AGENTS.md, а не CLAUDE.md.

     Важливо: Claude Code НЕ читає AGENTS.md автоматично — він читає тільки
     CLAUDE.md. Тому просте перейменування зламає Claude.

     Рекомендований підхід: Import pattern

     Claude Code підтримує @шлях/до/файлу всередині CLAUDE.md. Це дозволяє
     зробити AGENTS.md єдиним джерелом правди:

     1. Перемістити весь вміст CLAUDE.md → AGENTS.md (у кожному з 5 пакетів)
     2. Замінити CLAUDE.md на однорядковий файл-обгортку: @AGENTS.md

     Результат:
     - Інші інструменти читають AGENTS.md безпосередньо
     - Claude читає CLAUDE.md, який імпортує AGENTS.md → ті самі інструкції
     - Один файл для редагування (AGENTS.md), CLAUDE.md більше не змінюється

     Файли до зміни (5 пар)

     ┌────────────────┬─────────────────────────┬───────────────────────────────
     ┐
     │     Пакет      │      Поточний файл      │          Нові файли
     │
     ├────────────────┼─────────────────────────┼───────────────────────────────
     ┤
     │ корінь         │ CLAUDE.md               │ AGENTS.md (вміст) + CLAUDE.md
     │
     │                │                         │  (@AGENTS.md)
     │
     ├────────────────┼─────────────────────────┼───────────────────────────────
     ┤
     │ server/        │ server/CLAUDE.md        │ server/AGENTS.md +
     │
     │                │                         │ server/CLAUDE.md
     │
     ├────────────────┼─────────────────────────┼───────────────────────────────
     ┤
     │ client/        │ client/CLAUDE.md        │ client/AGENTS.md +
     │
     │                │                         │ client/CLAUDE.md
     │
     ├────────────────┼─────────────────────────┼───────────────────────────────
     ┤
     │ reviewer-core/ │ reviewer-core/CLAUDE.md │ reviewer-core/AGENTS.md +
     │
     │                │                         │ reviewer-core/CLAUDE.md
     │
     ├────────────────┼─────────────────────────┼───────────────────────────────
     ┤
     │ e2e/           │ e2e/CLAUDE.md           │ e2e/AGENTS.md + e2e/CLAUDE.md
     │
     └────────────────┴─────────────────────────┴───────────────────────────────
     ┘

     Кроки виконання

     1. Для кожного пакету:
       - Прочитати поточний CLAUDE.md
       - Створити AGENTS.md з тим самим вмістом
       - Перезаписати CLAUDE.md на @AGENTS.md
     2. Оновити .gitignore / .cursorignore якщо потрібно (зазвичай не потрібно)

     Верифікація

     Після змін:
     - Запустити нову Claude Code сесію в проєкті
     - Перевірити що Claude відповідає на "Start of session" протоколом (читає
     INSIGHTS.md і підтверджує 3 пункти) — це означає, що файл завантажено
     - Перевірити що інший інструмент (напр. Codex CLI) теж бачить інструкції з
     AGENTS.md
